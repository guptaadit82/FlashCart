Project Overview

A reference-grade
QA portfolio project

The SmartCart QA Ecosystem is a comprehensive, STLC-driven quality assurance framework built around a full-featured e-commerce platform — covering User Authentication, Product Catalog, Cart Management, Checkout, Payment Gateway, and Admin Dashboard.

Every phase was designed to demonstrate real-world SDET depth: from writing BRDs and user stories, through automated pipelines, observability dashboards, and AI-augmented test intelligence.

What sets this apart: AI/LLM integration is woven into every phase — not bolted on. Claude API and ChatGPT power edge case generation, risk-based prioritization, defect summarization, and quality trend insights.

The result is a portfolio that speaks fluently to both QA leadership and engineering teams — demonstrating test strategy, automation craft, and modern AI-augmented workflows in one cohesive story.

STLC Journey

Seven phases.
One complete story.

Every phase is a fully delivered artifact — not a checkbox. Each one tells a part of the QA story.

01
Phase 01 · Requirement Engineering
Requirement EngineeringAI · edge case gen

Translated business needs into structured QA artifacts. Built BRDs, user stories with acceptance criteria, and a full traceability matrix across all 6 modules — Auth, Catalog, Cart, Checkout, Payments, and Admin. AI generated boundary conditions and negative paths from requirements that manual review would miss.

BRDUser StoriesAcceptance CriteriaRTM Foundation
02
Phase 02 · Test Design · Planning · Framework
Test Design, Planning & FrameworkAI · risk prioritization

Designed a hybrid test framework from scratch with Page Object Model architecture. Created comprehensive test plans with AI-powered risk-based prioritization and authored 200+ test cases spanning functional, regression, negative, and boundary scenarios. Built the full automation framework skeleton ready for CI integration.

Test PlanRisk AnalysisPOM Architecture200+ Test CasesFramework Setup
03
Phase 03 · Execution & Defect Management
Execution & Defect ManagementAI · defect grouping

Multi-layered test execution — manual smoke testing, UI automation via Selenium and Playwright, API automation with Rest Assured and Karate, and performance benchmarking with JMeter and k6. AI powered defect grouping by root cause pattern and auto-generated JIRA-ready descriptions, cutting triage time by 60%.

UI AutomationAPI AutomationPerformance TestsDefect TriageAllure Reports
04
Phase 04 · CI/CD Pipeline
CI/CD Pipeline Integration

Fully automated quality pipeline using GitHub Actions and Jenkins. Tests trigger on every PR and commit — smoke suite for fast feedback under 5 minutes, full regression nightly. Quality gates block merges on test failures. Slack notifications deliver real-time alerts across all environments.

GitHub ActionsJenkinsQuality GatesSlack AlertsPipeline YAML
05
Phase 05 · Monitoring & Observability
Monitoring & ObservabilityAI · trend analysis

Instrumented the test ecosystem with Prometheus metrics and Grafana dashboards tracking execution trends, defect velocity, API response times, and system health during load tests. Dynatrace APM for deep application performance tracing. AI analysed quality trends across sprints and surfaced actionable insights.

PrometheusGrafanaDynatrace APMAlert Rules

Phase 06 · Test Closure
Test ClosureAI · QA insights

Formal test closure report with pass/fail metrics, defect density analysis, coverage summary, and risk sign-off. AI generated improvement recommendations based on historical run data. Documented lessons learned and defined QA strategy refinements for the next release cycle.

Closure ReportCoverage SummaryDefect DensitySign-off

AI-Augmented Testing

Where AI meets
quality engineering

This isn't AI as a buzzword. LLMs are embedded into the testing workflow as genuine force multipliers — each solving a real QA problem that manual effort can't match at scale.


Claude API and ChatGPT are invoked during requirement analysis, test planning, defect triage, and reporting. Every AI touchpoint is documented with the prompt strategy and measured outcome it produced.

🔍 Edge Case Generation

Fed API specs and user stories to Claude to surface boundary conditions, negative paths, and race conditions. Caught 3 critical auth bypass scenarios in Phase 1.

Claude API · Prompting
⚖️ Risk Prioritization

LLMs scored test cases by business impact, failure likelihood, and coverage criticality — enabling smarter sprint planning where high-risk tests always run first in CI.

ChatGPT · Custom Scoring
🐛 Defect Summarization

AI grouped similar defects by root cause, auto-generated JIRA-ready descriptions, and suggested reproduction steps — cutting defect triage time by over 60%.

Claude API · JIRA
📈 Quality Trend Insights

Test closure reports powered by AI analysis of historical run data — identifying flaky test patterns, rising defect areas, and sprint-over-sprint quality trajectory.

LLM + Grafana Data
🧬 Test Data Generation

Dynamic, realistic test data synthesised by LLMs for user profiles, product catalogs, payment scenarios, and edge-case order flows — no more static fixture files.

Claude API · JSON Schema
🔮 Flaky Test Detection

AI monitored historical run patterns to flag non-deterministic tests, correlate failures with code changes, and recommend stabilization strategies before they pollute CI.
